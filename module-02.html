Module 02 • Tools and permissions

🧰 Systems, Access, and Resource Map

Learn what each system controls and how to avoid using the wrong tool for the wrong decision.

WhatWhyWhat to useHow to use itSelf-check
Module progress:

Learning objectives

  • Match each scheduling system to its purpose.
  • List the access a new scheduler needs before training can be effective.
  • Explain why communication tools do not replace QGenda updates.
  • Use the resource map to find worksheets and reference files.

Why this matters

  • Schedulers cannot safely update schedules without the right system access. Missing access blocks practice, delays urgent changes, and leads to workarounds that create inconsistencies.
  • Each system has a distinct job. QGenda is the live operational record, WebExchange supports public-facing on-call schedules, Epic/worklists show workload pressure, and reconciliation files support payroll and fairness tracking.

What to use

  • QGenda login and scheduling permissions, including Settings/Staff/Profile access when profile setup is part of the role.
  • WebExchange edit access for public-facing on-call schedules such as NHWK, Peds, and NM call.
  • Epic/worklist access with the correct worklists configured.
  • Network/shared folder access for scheduling folders and current-quarter documents.
  • Tiger Connect scheduler role/group access for current-day and next-day urgent issues.
  • Payroll/reconciliation files if the scheduler documents extra shifts, sold shifts, or paid coverage.

How to use it

  1. Confirm QGenda access first because live schedule maintenance cannot happen without it.
  2. Confirm WebExchange access before learning NHWK, Peds, or NM call workflows.
  3. Confirm Epic/worklist access before learning daily counts or volume-pressure monitoring.
  4. Open the current shared folder and locate the current quarter folder, schedule documents to print, reconciliation folder, tumor board resources, and contact lists.
  5. Test Tiger Connect role access so urgent communication can be routed through the scheduler role rather than personal phones.
  6. Document any access gap immediately and do not use screenshots or copied files as substitutes for live systems.
