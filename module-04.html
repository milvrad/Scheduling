Module 04 • Request handling

🗓️ Requests, Time Off, and Communication

Learn how to handle routine requests, urgent requests, vacation days, work-from-home requests, no-evening requests, and post-publication changes.

WhatWhyWhat to useHow to use itSelf-check
Module progress:

Learning objectives

  • Explain why requests must be documented in QGenda with request-date notes.
  • Differentiate vacation weeks, individual days off, weekend-off requests, holidays, WFH, no-evening, early shift, and NHWK time off.
  • Use the correct communication channel for routine versus current-day/next-day issues.
  • Describe what changes after a schedule is published.

Why this matters

  • Requests affect coverage, fairness, contract obligations, and physician expectations. Documenting the request date in QGenda creates an auditable trail and prevents missed or duplicate handling.
  • Post-publication changes are discouraged because downstream teams, sites, and radiologists may already be working from the published schedule.

What to use

  • QGenda request workflow, Vacation Tally, Stats, notes, and published schedule view.
  • Email request documentation for routine requests.
  • Tiger Connect scheduler role for current-day and next-day issues.
  • Weekend Days Off Grid for weekend availability and requests.
  • Current holiday schedule and holiday history when assigning holiday coverage.
  • Paycom/contract/onboarding information when a request intersects with obligations or restrictions.

How to use it

  1. Review the request and identify the request type: vacation week, single day, weekend off, holiday, WFH, no evening, early shift, or NHWK time off.
  2. Check timing rules and limits before entering the request. For example, WFH and no-evening requests generally require three weeks’ notice.
  3. Save or retain the request documentation before entering the schedule.
  4. Enter the request in QGenda using the correct task or request workflow.
  5. Add a request-date note, such as “Requested on 05.18.26,” when the workflow calls for it.
  6. Check Stats or the relevant tally to confirm the request appears correctly.
  7. Reply to the radiologist confirming that the request has been noted, or explain the next step if the request cannot be processed yet.
  8. For current-day or next-day issues, use the urgent scheduler role and update QGenda immediately after coverage is secured.

Published schedule standard

Published schedule standard
Once the schedule is published in QGenda, changes are not encouraged. The radiologist generally must find coverage and the change must be approved by schedulers. Current/next-day items are handled urgently through the scheduler role.
